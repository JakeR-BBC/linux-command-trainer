Stage 1 — Data foundation
The commands.json file. Get the data model right first — everything else is built on top of it.
Stage 2 — Basic drill loop
A minimal UI: show a prompt, type an answer, get feedback. No frills, just working.
Stage 3 — Scoring & progression
Track correct/incorrect, build in basic spaced repetition so weak spots resurface.
Stage 4 — Modes
Recognition (multiple choice) → Recall (type it) → Scenario (what would you use here?)
Stage 5 — Polish & sharing
Category filters, progress summary, deploy config for GitHub Pages.
